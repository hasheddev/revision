cd - #cd into last working directory user was in(previous)

You create a directory: mkdir /home/user/documents/projects/beta

You want to enter that directory: cd !$ (Bash automatically expands this to cd /home/user/documents/projects/beta)
When you type !$ and hit Enter, Bash first shows you what the command expanded to, then executes it.

The "Last Word" Rule: It only grabs the final argument. If your previous command was cp file1.txt file2.txt, then !$ will be file2.txt.

In Bash, $_ is a special parameter that is very similar to !$, but with a key difference in how and where it works.
While !$ is a "History Expansion" feature (mostly for interactive typing), $_ is a variable that stores the last argument of the previous command.
The Key DifferencesFeature
!$ (History Expansion)$_ (Special Parameter)
Where it worksInteractive terminal only.Both terminal and inside scripts.
When it expandsExpands before the command runs.Expands during the command execution.
ContextBased on your command history.Based on the very last executed entity.
